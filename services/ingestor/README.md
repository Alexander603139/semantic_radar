# Ingestor — сервис сбора и очистки новостных статей

Сервис отвечает за первый слой архитектуры: парсинг новостных сайтов, извлечение основного текста, заголовка, даты публикации и сохранение в структурированном виде. Работает асинхронно, использует `httpx` для статичных страниц и `playwright` для динамических (с JavaScript), извлекает контент с помощью `readability-lxml` и `trafilatura`.

## Установка и запуск локально

### Требования
- Python 3.11+
- Установленный браузер Chromium для Playwright

### Шаги
1. Клонировать репозиторий.
2. Создать виртуальное окружение и активировать:
```bash
   python -m venv venv
   source venv/bin/activate  # или venv\Scripts\activate для Windows
```
3. Установить зависимости:
```bash
pip install -r services/ingestor/requirements.txt
```
4. Установить браузер Playwright:
```bash
playwright install chromium
```
5. Запустить сервис:
```bash
python -m services.ingestor.src.main
```
По умолчанию сервер запускается на порту 8001.

### Переменные окружения
- INGESTOR_PORT — порт для FastAPI (по умолчанию 8001)
- EMBEDDER_URL — URL для вызова embedder (по умолчанию http://embedder:8002/embed)
- EMBEDDER_MOCK — если True, вызов embedder заменяется заглушкой (по умолчанию True)
- SCHEDULE_CRON — строка cron для планировщика (по умолчанию "0 5 * * 0" — каждое воскресенье 5:00)

## API
- POST /run
Запускает парсинг для переданного списка сайтов. Возвращает task_id для отслеживания статуса.

Запрос:
```json
{
  "user_id": "test_user",
  "sources": ["https://lenta.ru", "https://ria.ru"],
  "limit": 5
}
```
Ответ:
```json
{
  "task_id": "uuid",
  "status": "started",
  "message": "Задача поставлена в очередь"
}
```

- GET /status/{task_id}
Возвращает статус выполнения задачи.

Ответ:
```json
{
  "task_id": "uuid",
  "status": "completed",  // pending, running, completed, failed
  "result": {
    "total_articles": 10,
    "sources_processed": 2
  },
  "error": null
}
```

- GET /health
Проверка работоспособности сервиса.

### Планировщик
При старте приложения автоматически запускается встроенный планировщик (на базе APScheduler), который выполняет задачу по расписанию, заданному в SCHEDULE_CRON. В текущей версии задача использует тестового пользователя и фиксированный список сайтов из config.SOURCES. В будущем планируется получать список сайтов и настройки для каждого пользователя из user_manager.

### Интеграция с embedder
После успешного парсинга и сохранения статей, ingestor вызывает embedder (через embedder_client) для векторизации текстов. Пока вызов заглушен (EMBEDDER_MOCK=True), но контракт (формат запроса/ответа) уже зафиксирован. При переходе на реальный embedder достаточно установить EMBEDDER_MOCK=False и указать корректный EMBEDDER_URL.

## Беклог (дальнейшие улучшения)

### Алгоритм сбора ссылок (get_article_links_from_html)
- Улучшить эвристики для динамических сайтов: использовать wait_for_selector для контейнера ссылок, анализировать структуру страницы.

- Добавить поддержку sitemap.xml как альтернативного источника ссылок.

- Реализовать адаптивную фильтрацию ссылок (по наличию даты, ID, паттернам /news/, /story/ и т.д.) без жёстких исключений.

### Масштабирование и отказоустойчивость
- Перейти на очередь задач (например, Redis + RQ или Celery) для распределения парсинга.

- Добавить повторные попытки с экспоненциальной задержкой для проблемных сайтов.

- Внедрить мониторинг (метрики в Prometheus) и алертинг.

### Поддержка пользователей
- Интеграция с user_manager: получение списка сайтов и лимитов по user_id.

- Поддержка расписания для каждого пользователя (хранение в БД, чтение при старте планировщика).

- Добавление эндпоинта /admin/run с расширенными возможностями для администратора.

### Оптимизация
- Кеширование результатов парсинга (например, чтобы не перепарсить одну и ту же статью повторно).

- Асинхронная запись на диск (сейчас синхронная).

- Уменьшение дублирования кода между fetch_html_httpx и fetch_html_playwright.

### Безопасность
- Добавить авторизацию через JWT (проверка токена через user_manager).

- Ограничение количества запросов (rate limiting) для защиты от перегрузки.

## Используемые технологии
- FastAPI — веб-фреймворк

- httpx — асинхронный HTTP-клиент

- Playwright — управление браузером для рендеринга JavaScript

- readability-lxml / trafilatura — извлечение основного контента из HTML

- Pydantic — валидация данных

- APScheduler — планировщик задач
